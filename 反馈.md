达梦(Dameng)的数据模型后台有两个，SYS和SYSDBA，SYS有133可查询表格，SYSDBA有6可查询表格，用sql语句查询会出现乱码，鐗圭鍚堥噾、澶嶅悎鏉愭枡、榛戣壊閲戝睘、楂樻俯鍚堥噾。
在 Metabase 的可视化报表里看到乱码，达梦数据库的设置里面没有“额外 JDBC 选项”的输入框。


数据在存进达梦数据库的那一瞬间，就已经是一堆乱码了。

为什么会这样？
你用 VSCode 写的 dameng_gen_data.sql 文件是 UTF-8 编码的（有“特种合金”等中文）。但是，刚才我们用 docker exec 执行 disql 导入数据时，Linux 容器默认的语言环境可能是纯英文（POSIX/C），disql 在读取文件时“水土不服”，把中文当成火星文读了进去，然后以火星文的状态永久保存在了硬盘上。 所以 Metabase 怎么查，查出来的都是原汁原味的火星文。

终极解决办法（强制 UTF-8 重新导入）：
我们需要在导入数据的命令里，强行加上 export LANG=zh_CN.UTF-8，让 disql 戴上正确的眼镜去读 SQL 文件。

请在你的终端里，执行这唯一一条命令（它包含了强制 UTF-8 环境、重新读取文件并自动 COMMIT）：

Bash
docker exec dm8 bash -c 'export LANG=zh_CN.UTF-8 && export LD_LIBRARY_PATH=/dm8/bin && (cat /tmp/dameng_gen_data.sql; echo "COMMIT;") | /dm8/bin/disql SYSDBA/Dameng123'
